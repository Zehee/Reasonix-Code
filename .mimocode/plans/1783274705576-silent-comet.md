# Plan: Port kimi-code Inline Plan Mode to Reasonix

## Goal

Replace Reasonix's left-right split screen plan mode with kimi-code's inline card approach. Plan content renders as a bordered box within the conversation flow; user actions use modal overlays (reusing existing components).

## Current Architecture (to remove)

- `App.tsx` splits screen into left (35%) + right `PlanPanel` (65%)
- `PlanPanel.tsx` (625 lines): header + body + footer, 8 modes
- `planPanelOpen` flag controls the split
- `PanelWidthCtx` (just added) constrains text width in left panel

## Target Architecture (kimi-code pattern)

```
Conversation Flow (single column, full width):
  ├── UserCard
  ├── StreamingCard (assistant reply)
  ├── PlanProposalCard (inline bordered box) ← plan content lives here
  ├── ToolCard (step progress)
  └── ...

Modal Overlay (when user action needed):
  ┌─────────────────────────────────┐
  │  Ready to build with this plan? │
  │  ▶ Approve                      │
  │    Refine                       │
  │    Revise                       │
  │    Cancel                       │
  └─────────────────────────────────┘
```

## Existing Components to Reuse (no new creation needed for modals)

Reasonix already has modal-overlay components that PlanPanel replaced. We bring them back:

| Component | File | Purpose |
|-----------|------|---------|
| `PlanConfirm` | `PlanConfirm.tsx` (242 lines) | Modal picker: Approve / Refine / Revise / Cancel. Uses `ApprovalCard` + `SingleSelect`. |
| `PlanCheckpointConfirm` | `PlanCheckpointConfirm.tsx` (92 lines) | Modal picker: Continue / Revise / Stop at step checkpoints. |
| `PlanRefineInput` | `PlanRefineInput.tsx` | Free-text input for refine/reject feedback. |
| `PlanReviseEditor` | `PlanReviseEditor.tsx` | Step-level plan editor for revise mode. |

| `SingleSelect` | `Select.tsx` | Keyboard-driven single-option picker. |
| `PlanStepList` | `PlanStepList.tsx` | Step list with status glyphs. |
| `MarkdownView` | `markdown-view.tsx` | Lightweight markdown renderer. |

## Files to Create

### 1. `src/cli/ui/cards/PlanProposalCard.tsx` — Inline plan proposal card

A new card that renders the plan as a bordered box within the conversation flow (kimi-code's `PlanBoxComponent` pattern).

```tsx
// Structure:
<Box flexDirection="column" borderStyle="round" borderColor={TONE.accent}
     paddingLeft={1} paddingRight={1} marginTop={1} width="100%">
  <Text bold color={TONE.accent}> plan </Text>
  <MarkdownView text={planBody} />
  {steps && <PlanStepList steps={steps} completedStepIds={completedStepIds} />}
</Box>
```

Props: `{ body: string; steps?: PlanStep[]; completedStepIds?: Set<string>; summary?: string }`

## Files to Modify

### 2. `src/cli/ui/App.tsx` — Remove split screen, integrate inline plan + modals

**Remove:**
- `planPanelOpen` variable and all `planPanelOpen ? ...` ternaries
- Left/right `<Box flexDirection="row">` split → change to `<Box flexDirection="column">`
- `<PlanPanel>` render

- When `pendingPlan` exists: render `<PlanProposalCard>` in the card stream (or suppress the card stream and show PlanConfirm modal directly)

**Key change in render tree:**
```tsx
// BEFORE (split screen):
<Box flexDirection="row">
  <Box width="35%"> {/* left panel */} </Box>
  {planPanelOpen ? <PlanPanel /> : null}
</Box>

// AFTER (single column + modals):
<Box flexDirection="column">
  <Box flexDirection="column" flexGrow={1}>
    <CardStream />
    <LiveActivityArea />
  </Box>
  {/* Plan modals render above composer, same as ShellConfirm/ChoiceConfirm */}
  {pendingPlan && !stagedInput ? <PlanConfirm ... /> : null}
  {stagedInput ? <PlanRefineInput ... /> : null}
  {pendingCheckpoint ? <PlanCheckpointConfirm ... /> : null}
  {pendingRevision ? <RevisionBody ... /> : null}
  {pendingReviseEditor ? <PlanReviseEditor ... /> : null}
  <ComposerArea />
</Box>
```
